## CiteFlow — вебсервіс для створення та керування бібліографічними посиланнями

## 1. Вступ

CiteFlow — це вебсервіс для автоматичного створення, редагування, копіювання, збереження та експорту бібліографічних посилань. Система розробляється на основі аналізу сервісу-прототипу YesCite. Основна проблема, яку вирішує продукт, — складність ручного оформлення джерел за різними стилями цитування.

## 2. Загальний опис

Основні блоки: введення джерела, визначення типу даних, генерація цитування, вибір стилю, ручне створення цитування, попередній перегляд, копіювання, експорт, реєстрація, авторизація, збереження цитувань і адміністрування.

Актори: незареєстрований користувач, зареєстрований користувач, адміністратор.

## 3. Функціональні вимоги

- **FR-01 [Must]:** Система повинна дозволяти користувачу вводити DOI, URL або ISBN у єдине поле введення.
- **FR-02 [Must]:** Система повинна автоматично визначати тип введених даних: DOI, URL або ISBN.
- **FR-03 [Must]:** Система повинна створювати бібліографічне посилання на основі введеного DOI.
- **FR-04 [Must]:** Система повинна створювати бібліографічне посилання на основі введеного URL.
- **FR-05 [Must]:** Система повинна створювати бібліографічне посилання на основі введеного ISBN.
- **FR-06 [Must]:** Система повинна дозволяти користувачу вибирати стиль цитування APA 7, MLA 9 або Chicago 17.
- **FR-07 [Must]:** Система повинна показувати попередній перегляд сформованого бібліографічного посилання.
- **FR-08 [Must]:** Система повинна дозволяти користувачу копіювати готове цитування у буфер обміну.
- **FR-09 [Must]:** Система повинна показувати повідомлення про помилку, якщо джерело не знайдено або введені дані некоректні.
- **FR-10 [Should]:** Система повинна дозволяти користувачу створювати цитування вручну шляхом заповнення полів про джерело.
- **FR-11 [Should]:** Система повинна дозволяти користувачу редагувати неповні або неправильні метадані перед копіюванням результату.
- **FR-12 [Should]:** Система повинна дозволяти експортувати цитування у формати TXT, BibTeX, RIS та JSON.
- **FR-13 [Should]:** Система повинна дозволяти користувачу зареєструватися та увійти до облікового запису.
- **FR-14 [Should]:** Система повинна дозволяти зареєстрованому користувачу зберігати цитування до обраного.
- **FR-15 [Should]:** Система повинна дозволяти зареєстрованому користувачу створювати проєкти та додавати до них цитування.
- **FR-16 [Could]:** Система може дозволяти пакетне створення декількох цитувань одночасно.
- **FR-17 [Could]:** Система може підтримувати додаткові стилі цитування, наприклад Harvard, IEEE або Vancouver.
- **FR-18 [Could]:** Система може дозволяти спільний доступ до проєкту для кількох користувачів.

## 4. Нефункціональні вимоги

### Продуктивність
- **NFR-P-01:** Час відповіді для відкриття основних сторінок системи не повинен перевищувати 1 секунду для 95% запитів.
- **NFR-P-02:** Час створення цитування через зовнішні джерела метаданих не повинен перевищувати 3 секунди для 95% запитів.
- **NFR-P-03:** Якщо обробка запиту триває довше 3 секунд, система повинна показати індикатор завантаження.
### Безпека
- **NFR-S-01:** Система повинна передавати всі дані користувачів через захищене HTTPS-з’єднання.
- **NFR-S-02:** Паролі зареєстрованих користувачів повинні зберігатися тільки у хешованому вигляді.
- **NFR-S-03:** Система не повинна передавати персональні дані користувача третім особам без згоди користувача.
- **NFR-S-04:** Система повинна дозволяти користувачу видалити власний обліковий запис і пов’язані з ним дані.
### Зручність використання
- **NFR-U-01:** Інтерфейс системи повинен бути зрозумілим для користувача з початковим рівнем технічної підготовки.
- **NFR-U-02:** Система повинна мати адаптивний інтерфейс для екранів смартфонів, планшетів, ноутбуків і настільних комп’ютерів.
- **NFR-U-03:** Основна дія створення цитування повинна бути доступна користувачу з головної сторінки без обов’язкової реєстрації.
- **NFR-U-04:** Повідомлення про помилки повинні бути зрозумілими та пояснювати, що саме потрібно виправити.
### Надійність
- **NFR-R-01:** Доступність системи повинна становити не менше 99% часу на місяць.
- **NFR-R-02:** Система повинна зберігати створені проєкти зареєстрованого користувача без втрати даних після виходу з акаунта.
- **NFR-R-03:** У разі недоступності зовнішнього API система повинна повідомити користувача про проблему та запропонувати ручне введення джерела.
### Масштабованість
- **NFR-M-01:** Система повинна підтримувати щонайменше 100 одночасних користувачів на початковому етапі.
- **NFR-M-02:** Архітектура системи повинна дозволяти збільшення кількості одночасних користувачів без повної переробки програмного забезпечення.
- **NFR-M-03:** Система повинна дозволяти додавання нових стилів цитування без зміни основної логіки користувацького інтерфейсу.

## 5. Обмеження

- **C-01:** Система повинна бути реалізована як вебсервіс, доступний через сучасний браузер.
- **C-02:** Клієнтська частина системи повинна підтримувати роботу на комп’ютерах, планшетах і смартфонах.
- **C-03:** Для навчального проєкту бюджет на розробку не передбачений.
- **C-04:** Дедлайн виконання визначається графіком практичної роботи ПЗ 01.
- **C-05:** Система повинна враховувати вимоги щодо захисту персональних даних користувачів.
- **C-06:** Система не повинна вигадувати відсутні дані про джерело, якщо зовнішні метадані неповні.
- **C-07:** Якість автоматично створеного цитування залежить від повноти та правильності даних, отриманих із зовнішніх джерел.
- **C-08:** Пакетне створення цитувань не є обов’язковим для першої версії системи.

## 6. Глосарій

- **Бібліографічне посилання** — оформлений запис про джерело інформації, яке використовується у навчальній, науковій або іншій роботі.
- **Цитування** — посилання на використане джерело відповідно до певного стилю оформлення.
- **DOI** — цифровий ідентифікатор наукової публікації або іншого цифрового об’єкта.
- **ISBN** — міжнародний стандартний номер книги.
- **URL** — вебадреса сторінки або ресурсу в інтернеті.
- **APA 7** — сьоме видання стилю цитування American Psychological Association.
- **MLA 9** — дев’яте видання стилю цитування Modern Language Association.
- **Chicago 17** — сімнадцяте видання стилю цитування Chicago Manual of Style.
- **Метадані** — структуровані дані про джерело: автор, назва, дата публікації, видавництво, журнал, DOI, ISBN або URL.
- **Проєкт** — набір збережених цитувань, об’єднаних користувачем для конкретної роботи.
- **Обране** — список цитувань, які користувач зберіг для швидкого доступу.
- **API** — інтерфейс для обміну даними між програмними системами.
- **MVP** — мінімальна життєздатна версія продукту з базовими функціями.
- **GDPR** — регламент Європейського Союзу щодо захисту персональних даних.
- **MoSCoW** — метод пріоритизації вимог: Must Have, Should Have, Could Have, Won’t Have.
